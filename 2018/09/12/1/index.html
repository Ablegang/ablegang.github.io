Hexo + github pages 简单搭建博客

Sep 12, 2018

hexo是一个基于nodejs的博客系统，它会对markdown进行解析，并生成html，同时还自带本地server，你可以直接通过命令行运行一个本地服务器，在本地浏览博客内容。

github pages是将git仓库中的html开放到web上访问的一个服务，它只是为没有服务器、没有域名的人提供一个静态网站支持的服务。

现在，基于这两个工具，简单地搭建一个博客出来。

创建仓库

首先你需要有github账号，如何注册账号，这里就不啰嗦了。

直接从创建仓库开始说起。

登录到github中，点击上图的“new repository”。

第二步，需要注意，repository name 这一项，一定要输入 username.github.io 这样的格式，其中，username，是指你的github用户名，你可以直接输入图中，owner 中的用户名。

设置仓库

创建后，点击setting选项，进入仓库设置页面

找到下面设置中的“Github Pages”，选择分支并设置，一般使用master分支

建立本地git和github的ssh key关联

这一步与搭建博客没有必然联系，主要是为了使本地仓库可以向github仓库中push数据。

安装hexo

当本地的git和github建立好ssh key的授权之后，现在可以在本地安装hexo了。

hexo基于nodejs，它的依赖通过npm进行管理，所以，在那之前，你需要先安装好node和npm。

另外，因为npm本身就是基于nodejs运行的，所以，在安装npm之前，就需要安装nodejs。

具体的安装步骤，请见node和npm安装步骤
