2018年工作回顾

Jan 4, 2019

2018 年已经过去，到现在才有时间做一番梳理和总结。

过去一年主要的问题，暴露在这么一些方面：

  • 工作流
  • 管理
  • 沟通

工作流

这一年，因为团队都是新搭建的，大家的水平、专业能力、工作能力，良莠不齐，各自有着自己的一套工作方法。

那么，在我认为，什么是工作流？

比如：

  • 如何讨论问题
  • 如何约定时间
  • 如何开会
  • 如何做分享
  • 如何跟进 bug
  • 如何汇报工作
  • 如何处理紧急问题
  • 如何部署项目
  • 如何发布版本
  • 如何发布代码
  • 如何记录开发中遇到的问题
  • 如何跟进合作同事的进度
  • 如何提出需求
  • 如何接收需求
  • 如何写文档
  • …

如上这些，都是具体的执行工作，工作流本是指具体的执行，而非抽象的问题，这里需要区分开。

比如：如何控制项目进度、如何保持共识，这些都是比较抽象的问题，细化出来，会延伸出一系列问题，所以，我将它们划分为学术和思维一类，而工作流，仅仅是指具体的执行流程。

统一的工作流，能够最大程度地统一团队成员的开发、排错等思维，一定程度地避免问题，让问题还未发生，就先避免。

比如，我曾遇到过这样的问题：

  • 因为不一样的部署流程，同事部署的项目，出问题时我排错成本很高，沟通成本也很高
  • 因为前期没有统一处理紧急问题的工作流，导致任何一个紧急问题都需要我临时制定处理方案，大家等着我分配任务
  • 因为没有统一的开会流程，导致每一次开会，总有人迟到，又总需要我提醒带笔记本
  • …

日常开发中，还会有各种各样其他的问题，我们无法预见，就算预见了，可能临时救火，之后又给忘了，于是如此反复。

所以，总得抽出一些时间，去思考现在的痛点，想想有什么办法解决。

那么，新的一年里，我需要为团队，制定一份统一的工作流，在方方面面规范团队成员的工作，以此减少因为信息不同步、工作流不统一而引发的风险和成本，并且减少彼此间的认知差异。

管理

过去一年中，我面临着以下管理问题：

  • 需求管理
  • 代码管理
  • 版本管理
  • 架构管理
  • 方案管理
  • UI 管理
  • 轮子管理
  • bug 管理
  • 文档管理
  • 其它文件资源管理
  • 账号与费用管理
  • 团队管理
  • …

日常工作中，还有更多琐碎需要管理，只是许多事情都隐藏在日常中，一时很难在现在想起来。

这些管理，每一项，都需要很用心、很耐心地去做，一丝不苟地做，才能让项目更稳健。

新的一年，我需要做好管理方案，在实际工作中，不断磨练，不断完善，构建我自己的管理模式。

沟通

这里，把沟通列为第三个要深思的事情。

日常工作中，如果细细去总结，我们都会发现，很多让我们痛苦的问题，如果做好到位的沟通，通常可以避免。

这一年，我见过太多这样的情况。

比如：

线上出现一个 bug ，有人愤懑地批判技术人员，为什么出现这么低级的错误？难道做完都不检查吗？这也太不上心了。

又比如：

新发布一个版本，有人一脸震惊地找到产品经理：这怎么是这样的？这不可能是这么做的啊，难道你让客户xxx吗？

于是，这一年，不论是我，还是我的部门，会在担心受怕和痛苦中度过，特别是一些具体的技术岗，因为这些情况发生的多了，怨声载道也是自然的事情。

因为发出指责的人，并未考虑到，对方的心理感受，他们没有考虑到一些客观因素和客观事实。

比如，第一个比如。

线上的确不允许出 bug，但你得有科学的配套，你不能说让一个人，1小时点完一卡车的硬币，还不允许出错，还没有工具…

道理上，确实是，点钱怎能出错？钱怎么可以点错？

但也得考虑到实际情况，你至少给人家一个电子称去称一下，或者点钞机，再不济，多给人点时间检查一下。

但如果这些配套都没有，出了问题就说人不上心，就说人犯了低级的错误，人家能不委屈么？

再比如说，第二个比如。

产品，本身处于一个版本一个版本迭代的过程中，并非一簇即就。

如果，你所指责的东西，在产品经理的视野里面，并不存在，并且你以「第二个比如」里的语气，去指责产品经理，那么，产品经理，同样会觉得委屈。

因为问题的本质，并不是你所表达的那样，好似产品经理设计欠妥，又或者是考虑欠周，而是你们之间的信息不对称，你所以为的那样，在产品经理各版本计划中，并不存在。

你不能说现在的人太玻璃心，而是，有些东西，水滴石穿，再坚强的人，时间久了，也会变的。

如果一个人，委屈久了，或者一个团队，委屈久了，他会有战斗力吗？

我观察到这些现象，于是，我不禁思考一个问题：怎样才能避免这方面的问题

不得不说，良好的团队沟通，是团队凝聚力的基石，没有基石，团队就不稳固。

比如上面我举的两个例子，其实大多数人看过去，都会从中感受到一股深深的恶意，浓浓的指责。

一次两次还好，但如果是十次，二十次，每一次呢？

日积月累之下，就算再好的关系，也会变得不信任，也会变得焦躁起来。

并且，技术人员很大层面的满足，来自于成就感，总在担心受怕中度过，会严重影响他们的成就感和安全感。

如果，团队里每个人的心里，都掐着一把火，这个团队就会浮躁不已，时刻有可能爆炸，如何做事？又如何共事？

星星之火，可以燎原。

如果小团队十几人，尚且出现这种情况，若是人员扩充，企业文化，可能够撑得起快速壮大的团队？

许多公司，都会死在效率和人力上。

如果我们能够换一种表达方式，换一个看待角度，能够充分互相理解，会不会让氛围，变得更和谐？

虽然我也说不出什么科学道理，但是，一个开心健康的工作环境，和谐的工作氛围，总是能为公司发展带来正方向的影响吧？

所以，充分理解，友善沟通，在团队合作中，显得尤为重要。

这一点，过去一年中，我曾经看到，却又没有思考，曾经感受，却又没有表达，如此反复，让团队一起受了委屈，我之过。

新的一年，我应该注重这个问题，这本也是企业文化的一个部分，虽然这可能是人事部门更应该去深入思考的问题，但我亦应该明确，我要让我的团队，更加健康，更有凝聚力的目标。

自勉

新的一年，路还很远。

过去一年所遇到的问题，应化作力量，让我走得更远。

还需努力，还需坚持。
